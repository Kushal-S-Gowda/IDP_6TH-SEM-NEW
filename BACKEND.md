# FloodSense Pro — Backend & Research Documentation

**Audience:** Evaluator / viva / paper reviewers — **backend code only** (not UI).

---

## 1. What this system does

Per-zone **urban flood early warning** for Bengaluru using:

1. **Live weather** — OpenWeatherMap 3-hour forecast API (not stored for training).
2. **Physics engine** — `ml/hydrology.py` (TWI, SCS-CN, HAND, AHP-FHI, Rational method, Manning).
3. **Machine learning** — XGBoost on 16 features (optional ensemble with physics).
4. **Lead time** — `ml/early_warning_horizon.py` (hours until forecast rain triggers HIGH+).
5. **Audit log** — `logs/prediction_log.csv` (operational traceability).

---

## 2. Critical clarification: logs ≠ training

| Data | Path | Purpose |
|------|------|---------|
| **Training dataset** | `data/processed/flood_dataset.csv` | Generated by `ml/preprocess.py` (Monte Carlo + hydrology labels). |
| **Prediction log** | `logs/prediction_log.csv` | Written on every live inference (`prediction_log.py`). **NOT fed back into training.** |
| **OWM forecast** | API at runtime | Consumed per request; summed into 6h/24h/7d rainfall. |

**Evaluator question:** *"Are you storing 3h rainfall and training on logs?"*  
**Answer:** We store 3h slots only inside the OWM API response at inference time. The CSV log records inputs/outputs for audit. Model weights come from `flood_dataset.csv`, whose labels are **FHI risk classes** from `ml/hydrology.py`, not from the log.

## 4. Equations implemented (`ml/hydrology.py`)

### 4.1 Topographic Wetness Index (TWI)

**Reference:** Beven & Kirkby (1979); Shahabi et al. (2020); Springer *Natural Hazards* (2025).

\[
\text{TWI} = \ln\left(\frac{a}{\tan\beta}\right)
\]

- \(a = 10 / (d_{\text{river}} + 0.1)\) — contributing area proxy [km²]  
- \(\beta = \arctan(|z - 920| / 50)\) — slope from plateau deviation  
- Urban correction: \(\text{TWI}' = \text{TWI} \times (1 + 0.3\phi)\), \(\phi = \min(0.9, N_{\text{pop}}/55000)\)

### 4.2 SCS Curve Number (USDA TR-55)

\[
S = \frac{25400}{\text{CN}} - 254 \quad [\text{mm}]
\]
\[
I_a = 0.2 S,\quad Q = \frac{(P - I_a)^2}{P + 0.8 S} \quad \text{if } P > I_a
\]

**AMC adjustment** on CN (5-day antecedent \(P_5\)):

- AMC-I (\(P_5 < 35\) mm): \(\text{CN} = \text{CN}_{II} / (2.281 - 0.01281 \cdot \text{CN}_{II})\)  
- AMC-III (\(P_5 > 53\) mm): \(\text{CN} = \text{CN}_{II} / (0.4036 + 0.00572 \cdot \text{CN}_{II})\)

CN from land use (population proxy): 92 impervious, 88 industrial, 85 residential, 74 open.

### 4.3 HAND (Height Above Nearest Drainage)

\[
\text{HAND} \approx (z - z_{\min}) \cdot \left(1 - e^{-1/(d_{\text{river}}+0.05)}\right)
\]

Normalized factor: \(f_{\text{hand}} = \text{clip}(1 - \text{HAND}/90,\ 0,\ 1)\).

### 4.4 Soil saturation

\[
\theta = 1 - \exp\left(-\frac{P_{7d}}{\text{FC}}\right),\quad \text{FC} = 480\ \text{mm}
\]

### 4.5 Flood Hazard Index (AHP)

**Reference:** Mohammed et al. (2024) GIS+AHP; MDPI Water (2025); Fuzzy-AHP+ML (JUSIFO 2025).

\[
\text{FHI} = \sum_{i=1}^{7} w_i f_i \cdot \left(1 + 0.25 \cdot \frac{Q}{P}\right)
\]

| Factor | Weight \(w_i\) |
|--------|----------------|
| Rainfall (IMD 24h bands) | 0.2818 |
| TWI | 0.2210 |
| HAND | 0.1682 |
| River proximity | 0.1356 |
| Soil \(\theta\) | 0.0987 |
| Flood history | 0.0612 |
| Population exposure | 0.0335 |

Risk class from FHI: LOW \(<0.25\), MEDIUM \(<0.45\), HIGH \(<0.65\), EXTREME \(\geq 0.65\).

### 4.6 Rational method (peak discharge)

**Reference:** Chow, Maidment & Mays (1988).

\[
Q_p = 0.278 \cdot C \cdot i \cdot A \quad [\text{m}^3/\text{s}]
\]

\(i\) = mm/hr, \(A\) = km², \(C\) = SCS runoff ratio.

### 4.7 Manning drainage capacity

\[
V = \frac{1}{n} R^{2/3} S^{1/2}
\]

Default urban drain: \(n=0.013\), \(R=0.25\) m, \(S=0.002\).

### 4.8 Dry-weather rule

If \(P_{24h} < 5\) mm (IMD light-rain lower band): **active risk = LOW**.  
TWI/FHI still reported as **susceptibility** (static vulnerability).

### 4.9 Hybrid ensemble (`predict.py`)

When XGBoost is loaded:

\[
\text{rc} = \text{round}\left(0.55 \cdot \text{rc}_{ML} + 0.45 \cdot \text{rc}_{FHI}\right)
\]

Capped in light rain (\(<20\) mm/24h) to avoid ML over-alerting.  
**Reference pattern:** AHP-ML hybrid (*Natural Hazards* 2025; Bekasi Fuzzy-AHP+RF/XGB 2024).

## 5. Machine learning pipeline

### Features (16)

**Base (10):** rainfall_24h, rainfall_7d, temperature, humidity, wind_speed, elevation, river_proximity, flood_history_freq, soil_saturation_idx, population_density  

**Engineered (6):** twi, scs_runoff_mm, scs_runoff_ratio, fhi, hand_m, soil_saturation_theta  

### Labels

`risk_level` = `fhi_to_risk_class(FHI)` from hydrology — **not** arbitrary +1/+2 rules.

### Retrain after code update

```bash
python ml/preprocess.py
python ml/train_model.py
```

Outputs: `ml/models/best_model.pkl`, `ml/models/scaler.pkl` (16 features).

## 6. Novelty vs literature (2024–2026)

| Aspect | Typical papers | FloodSense Pro |
|--------|----------------|----------------|
| Susceptibility | GIS raster TWI + AHP map | **Zone-level TWI + HAND + SCS** on live OWM rain |
| ML | AHP-RF or AHP-XGB offline | **Online ensemble** physics + XGBoost each refresh |
| Lead time | Radar nowcast 0–3 h | **Forecast-slot scan** 0–48 h per zone |
| Dry season false alerts | TWI alone → false MEDIUM | **Explicit dry rule** \(P_{24}<5\) mm |
| Traceability | Black-box score | **`HydrologyTrace` dataclass** + `/api/ml/hydrology` |
| Logs | Often conflated with training | **Separated** audit vs `flood_dataset.csv` |

**Publication angle:** *Hybrid AHP-hydrology engine with forecast-driven lead time and constrained ML fusion for urban early warning — demonstrated on Bengaluru stormwater/lake flood zones.*

## 7. Lead time (`ml/early_warning_horizon.py`)

For each zone, scan OWM 3h slots with rolling 24h rain → `predict_risk()`.  
**Lead time** = first slot index \(\times 3\) hours where risk \(\geq\) HIGH and \(P_{24h} \geq 5\) mm.  
**System lead** = minimum across zones.

Benchmarks (documented in API `benchmarks` key): IMD nowcast 0–3 h, CWC 6–24 h, BBMP manual 0–2 h, FloodSense 6–48 h (forecast-limited).
